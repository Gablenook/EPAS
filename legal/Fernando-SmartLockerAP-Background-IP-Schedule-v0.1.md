## 1. Intended Use of This Schedule

This schedule is intended to be attached to or incorporated into the later IP Ownership and Contribution Agreement between Kevin / Toren and Fernando / SmartLockerAP / SmartLocker, LLC.

The goal is to make clear that Fernando / SmartLockerAP owns its backend platform, backend software, hosting model, reusable development methods, APIs, database structures, administrative tools, and other pre-existing technology, and that collaboration with Toren does not transfer those assets to Toren unless expressly agreed in writing.

This schedule should be completed by Fernando / SmartLockerAP and reviewed by counsel before signature.

---

## 2. Ownership Statement

Fernando / SmartLockerAP owns or controls the Fernando / SmartLockerAP Background IP described in this schedule, subject only to any expressly identified third-party components, open-source components, customer-owned materials, employer-owned materials, or excluded materials listed in Section 13.

Unless expressly agreed in a signed written agreement:

- Fernando / SmartLockerAP Background IP remains owned by Fernando / SmartLockerAP.
- Access to Fernando / SmartLockerAP Background IP does not create ownership rights in Kevin, Toren, SmartLocker, LLC, Penco Products, customers, installers, vendors, channel partners, or other collaborators.
- Technical collaboration, troubleshooting, integration, code review, customer deployment, AI-assisted development, or commercial discussion does not create implied joint ownership.
- Licenses to Fernando / SmartLockerAP Background IP must be express, written, limited, and tied to the agreed commercial or operational purpose.

## 5. SmartLocker, LLC and SmartLockerAP, LLC Ownership Boundary

The parties should confirm the ownership boundary between SmartLocker, LLC and SmartLockerAP, LLC before final signature.

Known business understanding to document:

- **SmartLocker, LLC** sells or may sell locker products made by Penco Products or others.
- **SmartLockerAP, LLC** develops or controls the backend platform used to support the software solution.
- SmartLocker, LLC may request licenses from Kevin / Toren and Fernando / SmartLockerAP for software required to support customer deployments.
- SmartLockerAP may later serve as a licensing administrator or commercial packaging entity for a combined solution, but that role should not alter underlying ownership unless expressly agreed in writing.

Open items to confirm:

1. Which entity owns existing backend source code?
2. Which entity owns customer contracts, if any?
3. Which entity owns SmartLocker branding and sales materials?
4. Which entity will sign IP, confidentiality, license, and customer-facing agreements?
5. Which entity will collect and distribute recurring software license fees, if any?
6. Which entity will be responsible for backend hosting, uptime, support, and cybersecurity obligations?

## 8. Integration Work Product

Integration Work Product may include API contracts, payload definitions, field mappings, interface documents, test plans, sample requests, sample responses, integration diagrams, and other artifacts needed for Toren / EPAS edge systems and SmartLockerAP backend services to interoperate.

The recommended starting position is:

- Toren owns Toren-side implementation.
- SmartLockerAP owns SmartLockerAP-side implementation.
- Integration specifications may be used by both parties for the agreed customer solution.
- Neither party receives ownership of the other party's platform merely because the systems integrate.
- Neither party may use integration materials to clone, replace, or commercially bypass the other party's platform outside the agreed license structure.

The final agreement should decide whether Integration Work Product is authored-owned, jointly owned only as to the interface layer, cross-licensed, or assigned to one party with a defined license to the other.

## 11. Access and Use Restrictions

Unless expressly authorized in a signed written agreement, no collaborator may:

- copy SmartLockerAP source code or confidential documents for use outside the collaboration;
- reverse engineer, decompile, disassemble, inspect, reconstruct, or derive source code from SmartLockerAP services, APIs, binaries, repositories, or deployed systems;
- use SmartLockerAP confidential information to train, prompt, or instruct AI systems outside an approved data-handling process;
- use SmartLockerAP backend architecture, data model, API internals, documentation, or commercial strategy to build a competing backend platform;
- file patent applications based on SmartLockerAP Background IP without Fernando / SmartLockerAP authorization;
- represent SmartLockerAP Background IP as jointly owned, Toren-owned, Penco-owned, customer-owned, or SmartLocker, LLC-owned unless expressly documented;
- sublicense SmartLockerAP Background IP except under an express written license.

## 15. Plain-English Summary

Fernando / SmartLockerAP Background IP is the body of work Fernando / SmartLockerAP has created around the backend platform: backend services, APIs, database model, hosting methods, admin tools, reusable methods, templates, and backend platform know-how.

The purpose of this schedule is not to prevent collaboration.
